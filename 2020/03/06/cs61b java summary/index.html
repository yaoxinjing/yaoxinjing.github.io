一、the Golden rule of eqeal

first we should known that there are 9 types of variable in java

  • the first 8 types ist primitive types(byte,short,int,long….)
  • the 9th type is referencese type called Object(referencese may be mull)

the idea of the golden rule of equal is that every declared variable has a box to save the so called “bits”

for the first 8 types of variable(primitive variable)the binary equivalent of this value is stored in the box

for the Object the adress of the Object stored in the box(指针被存放在box中)，we can use the adress etc.pointer to find where the object is stored.

the golden rule

  • b=a copies the bits from a into b
  • Passing parameters copies the bits

note: for primitive variable afte copy a and b is independent etc. a changed and b still remain the old value.but for Object,afte copy and Pass the parameters the two object etc.a and b point to the same adress,a changed,b will been automatisch changed

二、 Decalaration and Instantiation of the Arrays

the notation to declare und instantiate the new Arrays:

  • crate a 64 bit box for storing an int array address(Decalaration)

  • creates a new object etc.Array

  • pute the address of the Array into the 64 bits box named X

the Arrays in java is also object.thats means that the variable x not stored the whole value of the Arrays but the address of the Array.(the address of the Array X ist the adress of X[0])

三、List

the simplest IntList:

to instantiate a list we can use folowing instruction in the main-function:
IntNode X= new IntNode(11,null);
IntNode X= new IntNode(12,X);
IntNode X=new IntNode(13,X);

we get a list with 3 element(13 12 11)

the useful methods

  1. Return the size of the list using recursion

for recursion we always need to think the basic case plus anorther situation.

  1. Return the size of the list using iteration

  1. get the item of the ith Object using the iteration

Destructive vs.Non-Destructive methods

  • Destructive methods means the methosd modify the original list according to the requirements of the title
  • Non-Destructive methods means the methods create a new list based on the requirements and the original list
