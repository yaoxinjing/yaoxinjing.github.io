DLList

Why we need a DLList

wenn we thinking about the SLList,we can find that it is easy to add a item into the end of the list by using a new global variable called last,which record the position of the last element in the list.

But wenn we want to delete a item in the end of the list,it’s not enough to only record the position of the end of the list,since we need also to find out out the second to last item and then set its next poiter to be null,Adding a secondToLast will not help either,becaus then we need to find the second to last item in order to make sure secondTolast item and last.this cost is really high.

Therefor we need DLList etc.Doubly linked Lists.Every Item in DLList not only have the next variable but also a variable called privious so that every item is able to track it’s successor and predecessor.With this beautiful characteristic we can easily to remove the front and back of a list in consttant time.

Two different way to implement the DLList

  1. the DLList with 2 Sentinel etc.one in the front and one in the back

  2. An alternate approach is to implement the list so that is is circular,with the front and back pointers sharing the same sentinel node.

一、the Golden rule of eqeal

first we should known that there are 9 types of variable in java

  • the first 8 types ist primitive types(byte,short,int,long….)
  • the 9th type is referencese type called Object(referencese may be mull)

the idea of the golden rule of equal is that every declared variable has a box to save the so called “bits”

for the first 8 types of variable(primitive variable)the binary equivalent of this value is stored in the box

for the Object the adress of the Object stored in the box(指针被存放在box中)，we can use the adress etc.pointer to find where the object is stored.

the golden rule

  • b=a copies the bits from a into b
  • Passing parameters copies the bits

note: for primitive variable afte copy a and b is independent etc. a changed and b still remain the old value.but for Object,afte copy and Pass the parameters the two object etc.a and b point to the same adress,a changed,b will been automatisch changed
